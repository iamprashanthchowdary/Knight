Knight

nginx traffic observability agent — APT repository

Knight tails your nginx access logs and turns every request into traffic analytics, failure drill-down reports, and webhook/email alerts. It observes only — it never blocks or modifies traffic. Source: iamprashanthchowdary/Knight.

Install

On any Ubuntu or Debian server:

curl -fsSL https://iamprashanthchowdary.github.io/Knight/install.sh | sh

This adds a GPG-signed APT repository, then installs the knight package. Review install.sh first if you'd rather not pipe straight to a shell.

What the installer does

  • imports the repository's signing key to /usr/share/keyrings/knight-archive-keyring.gpg
  • adds /etc/apt/sources.list.d/knight.list
  • runs apt-get install knight, which creates an unprivileged knight system user (in the adm group so it can read /var/log/nginx) and starts it as a systemd service

Configure

Point it at your nginx access log, then restart:

sudo nano /etc/knight/config.json
sudo systemctl restart knight
curl -s 127.0.0.1:8090/v1/overview

Update

sudo apt update && sudo apt upgrade

Uninstall

sudo apt remove knight   # keeps /etc/knight/config.json
sudo apt purge knight    # also removes config + the knight user
